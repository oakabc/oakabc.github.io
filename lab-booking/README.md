# ระบบจองคอมพิวเตอร์ห้อง Lab

เว็บแอปจองคอมพิวเตอร์ 56 เครื่อง แสดงผลผ่านมือถือ (สแกน QR code หน้าเครื่อง)

มี 2 เวอร์ชันในโปรเจกต์เดียว เลือกใช้อย่างใดอย่างหนึ่ง:

| | เวอร์ชัน Node (dev/ทดสอบ) | เวอร์ชัน GitHub Pages (แนะนำสำหรับใช้จริง) |
|---|---|---|
| หน้าเว็บ | `public/` + `server.js` | `docs/` บน GitHub Pages |
| API | Express (`server.js`) | Google Apps Script (`apps-script/Code.gs`) |
| ฐานข้อมูล | ไฟล์ `bookings.json` | **Google Sheets** (เปิดดู/ลบรายการจองใน Sheet ได้เลย) |
| ค่าใช้จ่าย | ต้องมีเครื่องรัน Node | ฟรีทั้งหมด ไม่ต้องมี server |

👉 วิธี deploy เวอร์ชัน GitHub Pages + Google Sheets: ดู [DEPLOY.md](DEPLOY.md)
👉 สเปคระบบฉบับเต็ม (กติกา, API, โมเดลข้อมูล, UI): ดู [SPECIFICATION.md](SPECIFICATION.md)

## กติกาการจอง
- จองได้เฉพาะ **วันศุกร์ 13:00–17:00 น.** (เปิดจองล่วงหน้า 4 สัปดาห์)
- ขั้นต่ำ **30 นาที** จองต่อเนื่องยาวได้จนถึง 17:00
- ระบบกันจองซ้อนทับต่อเครื่องอัตโนมัติ
- ได้รหัส 6 หลักไว้ยกเลิกการจอง (API `/api/cancel`)

## ผังห้อง
8 แถว × แถวละ 7 เครื่อง เว้นทางเดินกลาง: `[1,2,3 | ทางเดิน | 4,5,6,7]`,
`[8,9,10 | ทางเดิน | 11,12,13,14]` … จนถึงเครื่อง 56

## วิธีใช้งาน
```bash
npm install
npm start          # เปิดที่ http://localhost:3456
```

ขั้นตอนใช้งานของนักศึกษา: **สแกน QR → เห็นผังที่นั่งทั้งห้อง → เลือกเครื่องที่ว่าง → เลือกเวลา → ยืนยัน**

- หน้าหลัก `/` — ผังที่นั่ง + จอง (mobile-first)
- โปสเตอร์ QR `/qr` — QR เดียวลิงก์ไปหน้าผังที่นั่ง พิมพ์ติดหน้าห้อง (Ctrl/Cmd+P)
- หน้าผู้ดูแล `/admin.html` — ล็อกอินด้วย PIN แล้วปิด/เปิดเครื่อง (เช่น ส่งซ่อม),
  ดูรายการจองชื่อเต็ม, ยกเลิกการจอง — PIN ตั้งผ่าน env `ADMIN_PIN`
  (เวอร์ชัน Google Sheets ตั้งใน `ADMIN_PIN` ของ `Code.gs`) ค่าเริ่มต้น `000000` ต้องเปลี่ยนก่อนใช้จริง
- (เสริม) `/?seat=12` — ลิงก์เจาะจงเครื่อง เปิดแผงจองเครื่องนั้นทันที

> **สำคัญ:** QR ฝัง URL ตามโดเมนที่เปิดหน้า `/qr` ดังนั้นให้เปิดผ่านโดเมน/IP
> จริงที่นักศึกษาจะใช้ (เช่น `http://10.x.x.x:3456/qr`) ก่อนสั่งพิมพ์

## API
| Method | Path | ใช้ทำอะไร |
|---|---|---|
| GET | `/api/config` | ค่าคงที่ + รายการวันศุกร์ |
| GET | `/api/bookings?date=YYYY-MM-DD` | รายการจองของวัน (ชื่อถูกปิดบัง) |
| POST | `/api/bookings` | จอง `{date, seat, startSlot, endSlot, name}` |
| POST | `/api/cancel` | ยกเลิก `{id, code}` |

ข้อมูลเก็บในไฟล์ `bookings.json` (เหมาะกับห้องเดียว/เซิร์ฟเวอร์เดียว —
ถ้าจะสเกลหลายห้องแนะนำย้ายไป SQLite/PostgreSQL)
